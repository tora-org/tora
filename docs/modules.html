Inject

  • Inject(token: any): DecoratorParameter
    • Defined in src/core/annotation/common/inject.ts:26

    由于 Typescript 中具有值和类型双重身份的只有 Class 和 Enum。
    所以当你需要使用 Class 和 Enum 以外的值进行依赖查找时，可以使用此装饰器，显示指定一个非类型的值作为查找 token。

    比如使用一些特殊的字符串。

    在 typescript 中实现依赖注入时，用来查找依赖项的 token 需要满足条件：即是值，又是类型。

    在当前的 typescript 版本中（< 4.4.2），满足这个条件的概念只有 Class 和 Enum。

    要使用其他的值表示类型单纯通过 reflect-metadata 就做不到了。@Inject 就是一种辅助实现方式。

    Parameters

    • token: any

      任何可以通过 === 进行相等判断的值。一般会选择具有某些含义的字符串。

    Returns DecoratorParameter

KoaResponseType

KoaResponseType: string | Buffer | Stream | Object | any[] | null
  • Defined in src/http/__type__.ts:10

Koa 支持的响应体类型。

不同类型对应的处理方式如下：

  • string

    1. 直接写入响应流。

    2. Content-Type 会被默认设置为 text/html 或者 text/plain 同时携带 charset=utf-8。

    3. Content-Length 会被设置为对应数值。

    4. 响应状态码如果没有指定则会被设置为 200，包括空字符串的情况。

  • Buffer

    1. 直接写入响应流。

    2. Content-Type 会被默认设置为 application/octet-stream。

    3. Content-Length 会被设置为对应数值。

    4. 响应状态码如果没有指定则会被设置为 200。

  • Stream

    1. Content-Type 会被默认设置为 application/octet-stream。

    2. 当响应体被设置为 Stream 类型时，Koa 会自动添加一个 .onerror 监听器，用于捕获所有的 error 事件。

    3. 当请求连接断开（包括提前断开），Stream 会被销毁。

    4. 如果不需要这两个特性，则不要直接将响应体设置为 Stream 类型。

    5. 响应状态码如果没有指定则会被设置为 200。

  • Object 或者 Array

    1. 通过 JSON.stringify 进行序列化后写入响应流。

    2. Content-Type 会被默认设置为 application/json; charset=utf-8。

    3. Content-Length 会被设置为对应数值。

    4. 响应状态码如果没有指定则会被设置为 200。

  • null

    1. 响应体无内容。

    2. 响应状态码如果没有指定则会被设置为 204。
